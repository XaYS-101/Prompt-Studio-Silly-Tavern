# Prompt Studio

Расширение SillyTavern для тех, кто пишет, правит и тестирует промпт-пресеты Chat Completion. Оно заменяет тесное встроенное редактирование Prompt Manager полноэкранной студией: большой редактор с автодополнением и подсветкой макросов, живое превью со счётчиком токенов, история версий с diff, библиотека переиспользуемых блоков и редактор regex-скриптов с живым тестером.

**Ваши пресеты в безопасности.** Ровно как в родном Prompt Manager, правки сначала применяются к текущей сессии; файл пресета на диске меняется только по кнопке **«Обновить файл пресета»**. Снапшоты версий и блоки живут в собственных настройках расширения и не трогают файлы ST. Удалите расширение — и всё останется как было.

**Важно:** расширение активно развивается. Если что-то сломалось — откройте консоль браузера, поищите ошибки с префиксом `[PS]` и создайте issue с подробностями.

## Что умеет

### Студия
Полноэкранный редактор текущего пресета Chat Completion. Открывается кнопкой **«Открыть студию»** в настройках расширения (панель Extensions ➜ Prompt Studio).

Слева — список промптов в реальном порядке: чекбоксы включения, бейджи ролей, счётчики токенов, перестановка стрелками и **перетаскиванием** (на ПК). Справа — редактор выбранного промпта со всеми полями родной формы (название, роль, позиция, глубина, порядок, триггеры, запрет переопределений).

В шапке видно, какой пресет редактируется, и красный бейдж **«не сохранено в пресет»**, как только состояние сессии разошлось с файлом на диске. Один клик по **«Обновить файл пресета»** записывает его — через родное действие обновления ST.

### Редактор
- **Автодополнение макросов** — то же, что ST использует в своих полях: набираете `{{` и получаете подсказки с описаниями и аргументами.
- **Подсветка макросов** — `{{макросы}}` подкрашены прямо в тексте; семейство переменных (`{{getvar}}`, `{{setvar}}`, …) своим цветом, комментарии (`{{// …}}`) другим, а макросы, которых нет в реестре, помечаются красным — опечатки видны мгновенно.
- **Живое превью** — промпт с подставленными макросами по текущему открытому чату, плюс счётчик токенов, обновляется по мере набора.
- **Тестовая сборка** — строит полный промпт для активного чата (точный список сообщений, который ST отправил бы в API), ничего не отправляя: каждое сообщение с ролью и токенами, и общий итог.

### Два пресета бок о бок
Вкладка **«Два пресета»** ставит активный пресет рядом с любым другим пресетом Chat Completion: у каждого промпта бейдж статуса (одинаково / отличается / есть только в одном), а клик открывает оба текста рядом. Левый редактируется как обычно (живой пресет), правый пишется прямо в файл второго пресета (активный пресет так не трогается никогда). Кнопки копирования переносят текст в обе стороны, а встроенный diff показывает, что именно отличается.

### История версий
Снапшот промпта делается автоматически перед каждой сессией правок и вручную в любой момент (с заметкой). Вкладка «История» показывает все снапшоты текущего пресета с фильтром по промпту, восстановлением и удалением. **Diff с текущим текстом** показывает, где и что изменилось: номера строк (старая/новая), сводка +/−/~, пословная подсветка внутри изменённых строк, свёрнутые куски без изменений (клик раскрывает) и отдельно — смена названия/роли. Восстановление сначала снапшотит текущее состояние, так что оно всегда обратимо. Лимиты (на промпт / всего / по размеру), дедупликация и индикатор объёма не дают распухнуть `settings.json`.

### Библиотека блоков
Переиспользуемые фрагменты текста — стандартные секции инструкций, дисклеймеры, гайды по стилю. Блок создаётся с нуля или из выделенного текста в редакторе, а затем вставляется в позицию курсора любого промпта одним кликом. Макросы внутри блоков работают как обычно.

### Менеджер переменных
Отдельная вкладка для локальных (чатовых) и глобальных переменных (семейство `{{getvar}}` / `{{setvar}}`): создание переменной с именем, значением и областью (сразу открывается в редакторе), переименование, правка значений на месте (автосохранение), кнопки **+1 / −1** для числовых значений, удаление с подтверждением, поиск и фильтр по области — рассчитано на 50+ переменных (компактные строки, один скроллируемый список, живой счётчик). Каждая переменная раскрывается в список макросов с понятными подписями — `{{getvar}}`, `{{setvar}}`, `{{addvar}}`, `{{incvar}}`, `{{decvar}}` и шорткаты `{{.имя}}` / `{{$имя}}`, у каждого написано, что он реально делает — со вставкой в позицию курсора и копированием в один клик. Сворачиваемая справка «Как работают макросы переменных» разбирает самое запутанное: какие макросы выводят текст, а какие нет, и вложенность вроде `{{setvar::hp::{{roll:1d20}}}}`. А в редакторе промпта отдельная кнопка открывает пикер с поиском, который вставляет выбранную форму макроса прямо в позицию курсора — и там же можно создавать переменные, видеть, какие из них промпт уже использует (они помечены и подняты наверх), в один клик создавать упомянутые-но-несозданные и вставить сразу все переменные строками `имя: {{getvar::имя}}` для блока состояния.

### Справочник макросов и переменных
Каталог с поиском и группировкой по категориям — все макросы, которые реально зарегистрированы в вашей сборке ST (строится по живому реестру, кастомные макросы расширений тоже видны), с описаниями, аргументами, синонимами и примерами. Один клик вставляет макрос в редактор. Ненужные макросы можно скрыть из списка кнопкой-глазом — тумблер «Показать скрытые» в любой момент вернёт их. ### Regex-скрипты
Вкладка Regex редактирует глобальные скрипты встроенного расширения Regex со всеми их полями и добавляет то, чего нет в родном редакторе:
- **индикатор корректности** регулярки,
- **живой тестер** — вставьте образец, увидьте подсвеченные совпадения и результат после замены, посчитанный родным движком ST (так что trim/макросы/флаги ведут себя ровно как в бою),
- встроенная **шпаргалка по regex** (якоря, классы, квантификаторы, группы, lookaround, флаги и объяснение ST-специфичных полей),
- **библиотека шаблонов** — 36 готовых скриптов в каталоге с поиском, добавление в один клик:
  - *Очистка* (13): вырезать OOC-комментарии, убрать HTML, удалить блоки `<think>`, схлопнуть пустые строки и двойные пробелы, выпрямить «умные» кавычки, убрать звёздочки / markdown-заголовки / эмодзи, распаковать кодовые блоки, подчистить концы строк, нормализовать многоточия;
  - *Форматирование* (3): срезать префикс `{{char}}:` в начале строк (с подстановкой макроса), «ёлочки» в диалогах, тире вместо `--`;
  - *Оформление* (18, только отображение): строки `Погода:` / `Локация:` / `Время:` / `Инвентарь:` / `Задание:` в рамках, плашки `Настроение:` / HP / броска кубика, панель ` ```status `, пузырь ` ```sms `, лист письма ` ```letter `, терминальная плашка `[System]`, стилизация `%%мыслей%%` и `((шёпота))`, раскрывашка `||спойлер||`, разделители сцен, заголовки глав, строки с 🎵;
  - *Промпт / экономия токенов* (2, `promptOnly`): эмодзи не попадают в промпт, старые блоки ` ```status ` вырезаются из истории промпта.

  Визуальные шаблоны работают через `markdownOnly` — текст чата не меняется, а рамки собраны на CSS-переменных темы ST и подстраиваются под любую тему.

Скрипты персонажей и пресетов показываются только для чтения.

### Мелочи
- Английский и русский интерфейс (по умолчанию русский; «Авто» следует за языком ST).
- Экспорт и импорт данных расширения (снапшоты, блоки, лимиты) в JSON.
- Кнопка **«Очистить все данные»**, сбрасывающая только данные самого расширения. Пресеты, промпты и regex-скрипты принадлежат ST и не трогаются.

## Установка

В SillyTavern откройте **Extensions ➜ Install extension** и вставьте адрес репозитория:

```
https://github.com/XaYS-101/Prompt-Studio-Silly-Tavern
```

Ни серверного плагина, ни правок `config.yaml` — расширение целиком работает в браузере.

Собрано и проверено на SillyTavern 1.17.0. На старых версиях расширение деградирует аккуратно: функции, чьих API нет (реестр макросов, автодополнение, движок regex), прячутся, а не ломают всё.

## Примечания

- Редактор промптов работает с пресетами **Chat Completion** (те, что в Prompt Manager). Шаблоны instruct/system prompt для Text Completion — вне охвата.
- Маркеры (Chat History, Char Description, …) — структурные заглушки: их можно включать и переставлять, но содержимое подставляет сам ST.
- Тестовой сборке нужен открытый чат с персонажем и рабочее подключение Chat Completion — она использует родной dry-run проход генерации ST.
